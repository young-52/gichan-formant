# 그리기 모드 기획안 (2차 확정안)

> 단일 플롯(일반 모드)에서만 구현. 다중 플롯은 단일에서 완벽히 동작한 뒤 별도로 검토.

---

## 1. 개요

- **목적**: 연구자들이 Formant Plot 위에 선·영역·참조선을 그려 모음 공간·diphthong 등을 표시.
- **범위**: 선(Polyline), 영역(Polygon), 참조선(Reference line) 세 가지 그리기 유형.
- **진입**
  - 캔버스 **좌측 하단**에 그리기 모드 인디케이터(버튼) 배치. 기존 캔버스 **우측 상단**의 모드 인디케이터(tool_ruler, 라벨 이동 등)와 동일한 스타일로, **캔버스 안에** 버튼이 생김.
  - **선 / 영역 / 수평 참조선 / 수직 참조선** 등 버튼을 누를 때마다 그때그때 모드 전환.
  - **배치 제약**: 기존 영역(캔버스 길이, 프로그램 가로 길이 등)이 **절대로** 바뀌지 않도록 매우 조심하여 배치.
- **기존 도구와의 관계**
  - 그리기 모드와 기존 분석 도구(tool_ruler, 라벨 이동 등)는 **반드시 상호 배타**.
  - 그리기 모드 진입 시 기존 도구는 꺼지며, 그동안 진행 중이던 작업(예: 눈금자로 그리던 것)은 자연스럽게 사라지는 것이 올바른 동작.

## 2. 그리기 유형

### 2.1 선 (Line / Polyline)

- **동작**
  - 캔버스 클릭으로 점 추가. 점과 점 사이는 직선으로 연결.
  - **z-order**: 선은 점 마커 **뒤**에 그려짐(시각적으로 선이 점 마커에 가려짐).
  - 데이터 좌표(F1-F2, nF1-nF2, F1-F3 등)로 저장 → 축 범위/스케일 변경 시에도 비율 유지.
- **스냅**
  - 눈금자 툴과 동일: centroid·raw 점 호버 시 스냅, 툴팁에 좌표 표시. **기존 눈금자 툴 코드에서 스냅·호버·툴팁 로직을 import하여 재사용.**
  - **스냅 끄기 옵션 없음.** 자유 좌표에 점을 두는 것은 지원하지 않음.
  - tool_ruler에서는 호버 시 동그라미/사각형이 결과물로도 남지만, 그리기 모드의 선·영역에서는 호버 시 생기는 마커는 **결과물로 남지 않음** (스냅 피드백만).
- **완료**
  - **Enter** 키 또는 **더블클릭**으로 현재 선 완료. 완료 시 레이어에 객체로 등록.
- **취소**
  - **Esc**: 지금 그리던 선 전체 취소(점 0개로 초기화).
  - **Ctrl+Z**: 점 하나만 롤백. **그리기 모드 내에서만** 동작하며, **현재 그리는 해당 객체**에만 적용.
- **화살표**
  - 선을 그린 뒤 완료 → **디자인**에서 "화살표" 스타일 적용.
  - "끝에만 화살표" / "선분마다 화살표"는 사용자가 디자인에서 선택.
- **레이어 이름**
  - 자동 제안: 꼭지점이 centroid에 스냅됐으면 `"선 : a-b-c"`, 아니면 `"선 1"` 등.
  - 레이어 이름 수정: 라벨·그리기 공통 기능으로 추가(아래 §3 참조).

### 2.2 영역 (Area / Polygon)

- **동작**
  - 선과 동일하게 클릭으로 점 추가. 점과 점은 직선으로 연결.
  - **닫기**: **시작점 근처에 마우스를 호버**하면 기존 스냅과 같이 동그라미/사각형이 뜨며 가까이 있음을 알려줌. **그 범위에서 클릭**하면 마지막 점이 시작점과 연결되어 폐다각형이 완성됨.
  - "영역 닫기" 버튼은 없음. **시작점을 다시 클릭(스냅 범위 내)하지 않는 한 영역은 닫히지 않음.**
  - **닫힌 순간** 자동으로 레이어에 객체 등록(별도 완료 키 없음).
- **취소**
  - **Esc**: 현재 그리던 **영역 전체** 취소.
  - **Ctrl+Z**: 점 하나 롤백. 그리기 모드 내에서만, 해당 객체 대상.
- **채우기**
  - 닫힌 순간 옅은 알파로 채우기(라벨 영역 디자인과 유사). 테두리/채우기 색·스타일은 그리기 디자인에서.
- **레이어 이름**
  - `"영역 : a-b-c-d"` 또는 `"영역 1"` + 사용자 편집.
- **영역 넓이**
  - **로직 필수**: 그린 폴리곤의 넓이를 데이터 좌표 기준으로 계산하는 로직 반드시 구현.
  - **표시**: 라벨의 디자인 패널처럼 **세부 표시 항목**으로 넓이 표시를 **켜고 끌 수 있음**. tool_ruler의 거리 표시 박스처럼 **드래그로 위치 이동 가능**. **해당 그리기 레이어가 포커스일 때만** 넓이 표시/이동 가능.

### 2.3 참조선 (Reference line)

- **입력 방식**: 축과 평행만. **수평 참조선**과 **수직 참조선**을 **별도 모드(버튼)**로 둠.
- **동작**
  - 수평 또는 수직 모드 선택 시, 마우스 **호버 위치**를 따라 **연한 preview 선**이 미리 보임.
  - **축에 해당 숫자 표시**. 현재는 포먼트 단위만 있으며 단위가 비교적 크므로, **10 단위 정수만** 표시(예: 500, 510, 520 …).
  - **클릭**하면 해당 위치에 참조선이 확정되어 레이어에 등록.
- **대각선**: 1차에서는 지원하지 않음.
- **레이어 이름**
  - `"참조선 : F1=500"`, `"참조선 : F2=1500"` 등 + 사용자 편집.

## 3. 레이어 연동

- **그리기 탭** (이미 레이어 도크에 "그리기" 탭 존재)
  - **상단**: 그리기 공통·개별 디자인(선 색/두께/스타일, 영역 채우기, 화살표 유무·위치 등).
  - **목록**: 그린 객체 순서대로 — "선 : a-b-c", "영역 : i-a-u", "참조선 : F1=500" …
  - 각 행: 눈(가시성), 이름, 순서 변경, 삭제, (선택) 잠금.
- **레이어 이름 수정**
  - **포토샵 로직**: 이름 쪽을 **더블클릭**하면 편집 가능. 라벨·그리기 레이어 공통.
- **이동**
  - 선·영역은 **데이터 좌표(포인트) 기반**으로 그려지므로, **어떤 경우에도 이동 불가**. 이동 기능은 제공하지 않음.
- **잠금**
  - 잠금 시 **삭제 불가**.

## 4. 키·동작 정리

| 동작           | 선                 | 영역                     | 참조선     |
|----------------|--------------------|--------------------------|------------|
| 완료           | **Enter** / 더블클릭 | 시작점 클릭 시 **자동**   | 클릭 시 확정 |
| 취소(전체)     | **Esc**            | Esc                      | Esc        |
| 점 롤백        | **Ctrl+Z** (해당 객체만, 그리기 모드 내) | Ctrl+Z (해당 객체만, 그리기 모드 내) | -          |

## 5. 데이터·좌표

- **저장 단위**
  - 객체 하나 = 선 하나 / 영역 하나 / 참조선 하나.
- **공통 필드(안)**
  - `type`: `"line"` | `"polygon"` | `"reference"`
  - `name`: 표시 이름.
  - `visible`: bool.
  - `order`: 정수(z-order 또는 목록 순서).
  - **(필수) 단위 메타데이터**: 어떤 작업이든 단위 메타데이터는 무조건 저장. 기존 코드에 메타데이터를 저장하지 않는 경우가 있으면 무조건 추가.
  - (선/영역) `points`: 데이터 좌표 배열 `[[x,y], ...]`.
  - (참조선) `mode`: `"horizontal"` | `"vertical"`, 및 값(축 위치).
  - **스타일**: 색, 두께, 선 스타일, 화살표(끝만/선마다), 채우기(영역) 등 — 그리기 디자인 연동.
- **좌표계**
  - 모든 점은 **데이터 좌표**(F1, F2 또는 nF1, nF2)로 저장. 축 스케일 변경 시에도 일관되게 표시.

## 6. 코드 구조 제안

- **폴더**
  - 프로젝트 내 `draw/` (또는 `GichanFormant/draw/`) 생성.
- **모듈 분리**
  - `draw_line.py` — 선(폴리라인) 그리기·저장·표시 로직.
  - `draw_polygon.py` — 영역(폴리곤) 그리기·저장·표시·**넓이 계산** 로직.
  - `draw_reference.py` — 참조선 그리기·저장·표시 로직.
  - `draw_common.py` 또는 `draw_utils.py` — 좌표 변환, **기존 눈금자 툴의 스냅·호버·툴팁 import 및 재사용**, 데이터 모델 공통 부분.
- 그리기 관련 기능은 위 폴더·별도 py로 유지하고, 레이어·캔버스와의 연동만 UI에서 처리.

---

## 7. 영역 넓이

- **요구**
  - 영역(폴리곤)의 넓이를 **데이터 좌표 기준**으로 계산하는 로직 반드시 구현.
- **표시**
  - 라벨 디자인의 세부 표시처럼 **켜기/끄기 가능**.
  - tool_ruler의 거리 표시 박스처럼 **드래그로 위치 이동 가능**.
  - **해당 그리기 레이어가 포커스일 때만** 넓이 표시 및 이동 가능.

---

## 8. 추가로 필요한 작업(참고)

- **레이어 이름 수정**: 라벨·그리기 공통 — 목록에서 이름 **더블클릭 시 편집** (포토샵 스타일).
- **그리기 디자인 패널**: 선 스타일, 화살표(끝만/선마다), 영역 채우기 등 UI 설계·연동은 기획 확정 후 진행.
- **단위 메타데이터**: 저장하지 않는 기존 코드가 있으면 무조건 추가.

---

## 9. 미정·추후 결정

- 참조선: 대각선 지원 여부(현재는 미지원).
- 영역 넓이 단위 표기(Hz², nF² 등) 세부 규칙.
- 그리기 객체의 프로젝트/세션 저장 포맷 세부.
